# Arabic Sign Language Recognition

A deep-learning image classification project designed to recognise Arabic Sign Language alphabet gestures from uploaded images.

The project trains and compares three transfer-learning architectures:

* ResNet18
* MobileNetV3 Small
* EfficientNet-B0

A Streamlit application provides a simple interface where users can upload a hand-sign image and view predictions generated by the trained models.

## Project Overview

Arabic Sign Language recognition can help reduce communication barriers between sign-language users and people who do not understand signing.

This project explores the use of convolutional neural networks and transfer learning to classify static Arabic hand signs. Three pretrained computer-vision models are adapted for a 32-class classification task and evaluated through separate training notebooks.

The trained models can then be loaded into a Streamlit application for interactive image prediction.

## Features

## Models

### ResNet18

ResNet18 uses residual connections to support effective deep-network training while maintaining a relatively compact architecture.

Model weight filename:

```text
resnet18.pth
```

### MobileNetV3 Small

MobileNetV3 Small is a lightweight architecture designed for efficient inference and deployment on devices with limited computational resources.

Model weight filename:

```text
MobileNet.pth
```

### EfficientNet-B0

EfficientNet-B0 balances model depth, width and image resolution to provide strong classification performance with efficient resource usage.

Model weight filename:

```text
EfficientNet.pth
```

## Image Preprocessing

Before prediction, each uploaded image is:

1. Resized to `224 × 224` pixels.
2. Converted into a PyTorch tensor.
3. Normalised using ImageNet mean and standard-deviation values.
4. Moved to the available CPU or GPU device.
5. Passed through each trained classification model.

The predicted class is selected from the model output with the highest score.


The filenames must match the names used inside `DemoLast.py`.

Large model files can be distributed through:

* GitHub Releases
* Git Large File Storage
* Google Drive
* Hugging Face
* OneDrive

### Uploading the Weights with Git LFS

The current `.gitignore` excludes `.pth` files. Remove the following line before using Git LFS:

```gitignore
*.pth
```

Then run:

```bash
git lfs install
git lfs track "*.pth"
git add .gitattributes
git add resnet18.pth MobileNet.pth EfficientNet.pth
git commit -m "Add trained model weights using Git LFS"
git push
```

## Using the Application

1. Start the Streamlit application.
2. Select **Choose a JPG image**.
3. Upload an image containing an Arabic hand sign.
4. Wait for the image to be processed.
5. Review the predictions produced by:

   * ResNet18
   * MobileNetV3 Small
   * EfficientNet-B0

## Important Code Correction

In the EfficientNet prediction section of `DemoLast.py`, ensure that the EfficientNet model is called:

```python
outputs = model_EfficientNet(img)
```

It should not call:

```python
outputs = model_mobilenet(img)
```

Without this correction, the application displays the MobileNet prediction twice instead of producing an EfficientNet prediction.

## Training the Models

The repository provides a separate notebook for each model:

```text
Resnet18.ipynb
MobileNet_test.ipynb
EfficientNet.ipynb
```

To retrain a model:

1. Open the relevant notebook in Jupyter Notebook or Google Colab.
2. Configure the dataset path.
3. Confirm the class-folder structure.
4. Run the preprocessing and data-loading cells.
5. Train and evaluate the model.
6. Save the trained state dictionary using the expected filename.
7. Place the saved `.pth` file beside `DemoLast.py`.

## Dataset Structure

A standard image-folder structure can be used:

```text
dataset/
├── train/
│   ├── class_01/
│   ├── class_02/
│   └── ...
├── validation/
│   ├── class_01/
│   ├── class_02/
│   └── ...
└── test/
    ├── class_01/
    ├── class_02/
    └── ...
```

Each folder should contain images belonging to one Arabic Sign Language class.

The class order used during training must remain consistent with the class-label order used by the Streamlit application.

## Current Limitations

* The application accepts static JPG images rather than live video.
* Trained model weights must be supplied separately.
* Predictions are displayed independently without confidence scores.
* The application does not yet combine the three models into an ensemble result.
* Performance may vary depending on lighting, image quality, background and hand positioning.
* The class-label naming should be reviewed and standardised.
* The application currently has limited input validation and error handling.
